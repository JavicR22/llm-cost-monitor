### ⚠️ Problemas encontrados

1. **`alembic upgrade head` fallaba** — `DATABASE_URL` no disponible en el proceso de alembic
2. **"Provider 'openai' is not supported or inactive"** — tabla `providers` vacía, sin seeds
3. **TypeError: 'label' is invalid keyword for ProviderAPIKey** — columna `label` no existía en el modelo
4. **Delete key no desaparecía del UI** — `mutate()` re-fetch traía key con `is_active=false` aún visible
5. **"Provider 'google' is not supported or inactive"** — seed inicial puso google con `is_active=false`
6. **Dashboard mostraba $0.00** — `gemini-2.5-flash` no estaba en tabla `models` → `_write_log` silencioso
7. **`v.toFixed is not a function` en SpendChart** — Pydantic v2 serializa `Decimal` como string en JSON

### 🔧 Cómo se resolvieron

1. **Alembic env.py** — Agregado `load_dotenv()` con python-dotenv en `alembic/env.py` (try/except para robustez)

2. **Seed migration** (`b2c3d4e5f6a7`) — Inserta los 4 providers (openai, anthropic, google, mistral) y ~20 modelos con precios por token. UUIDs fijos para idempotencia.

3. **Label column** — Migration `c3d4e5f6a7b8` agrega `label VARCHAR(100) NULLABLE` a `provider_api_keys`. Modelo SQLAlchemy actualizado.

4. **SWR optimistic update** en `ServiceKeysList.tsx` y `ProviderKeysList.tsx`:
   ```ts
   mutate((current) => current?.filter((k) => k.id !== id), { revalidate: false })
   ```

5. **Activate google migration** (`d4e5f6a7b8c9`) — Upsert idempotente con `ON CONFLICT (name) DO UPDATE SET is_active=true`

6. **Missing Gemini models** (`e5f6a7b8c9d0`) — Agrega `gemini-2.5-flash`, `gemini-2.5-pro`, variantes `gpt-4o-2024-11-20`, `gpt-4.1`, `claude-3-7-sonnet-20250219`. Helper `_insert_model` revisa existencia antes de insertar.

7. **Decimal → float** en `backend/app/schemas/dashboard.py` (fix raíz) + coerción `Number()` en todos los helpers `fmtUsd`/`fmtNum`/`fmtTokens` del frontend (fix defensivo).

#### Google Gemini implementado

- Nuevo router: `backend/app/api/proxy/google.py` (`POST /v1/google/chat/completions`)
